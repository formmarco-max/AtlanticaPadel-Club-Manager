# Project Charter

# AtlanticaPadel Club Manager (APCM)

## 1. Project Overview

O AtlanticaPadel Club Manager (APCM) é uma plataforma web desenvolvida no âmbito do Projeto Final da Licenciatura em Gestão de Sistemas e Computação da Universidade Atlântica.

O projeto pretende disponibilizar uma solução moderna para a gestão integrada de academias de pádel, centralizando processos administrativos e operacionais numa única plataforma.

A aplicação permitirá gerir reservas de campos, sócios, treinadores, aulas e indicadores de gestão, proporcionando uma experiência simples, intuitiva e eficiente para todos os intervenientes.

---

## 2. Business Problem

O crescimento da modalidade de pádel em Portugal e na Europa levou ao aparecimento de inúmeras academias e clubes.

Muitas destas organizações recorrem a várias ferramentas distintas para gerir reservas, sócios, treinadores e atividades, originando processos pouco eficientes, duplicação de informação e dificuldades no acompanhamento da atividade diária.

Existe assim a oportunidade de desenvolver uma plataforma integrada que simplifique a gestão operacional de uma academia.

---

## 3. Project Objectives

### Objetivo Geral

Desenvolver uma plataforma web para gestão integrada de academias de pádel.

### Objetivos Específicos

- Gerir sócios
- Gerir treinadores
- Gerir reservas de campos
- Gerir aulas
- Disponibilizar um dashboard administrativo
- Implementar autenticação de utilizadores
- Desenvolver uma interface moderna e intuitiva

---

## 4. Project Scope

O projeto inclui:

- Gestão de utilizadores
- Gestão de sócios
- Gestão de treinadores
- Gestão de campos
- Gestão de reservas
- Gestão de aulas
- Dashboard administrativo
- Sistema de autenticação

Não fazem parte do âmbito desta versão:

- Aplicação mobile nativa
- Integração com pagamentos online
- Integração com equipamentos de controlo de acessos
- Sistema de videovigilância
- Inteligência Artificial para análise de jogos

Estas funcionalidades poderão ser consideradas em futuras versões da plataforma.

## 6. Success Criteria

O projeto será considerado bem sucedido caso:

- Todos os requisitos do MVP sejam implementados.
- A aplicação permita gerir uma academia de pádel de forma funcional.
- O sistema apresente uma interface intuitiva.
- A arquitetura seja modular e escalável.
- O relatório final documente adequadamente todas as fases do desenvolvimento.

---

## 7. Constraints

- Tempo disponível até ao final de julho.
- Desenvolvimento realizado por um único estudante.
- Recursos financeiros reduzidos.
- Desenvolvimento realizado para fins académicos.

---

## 8. Risks

| Risco | Impacto | Mitigação |
|---------|----------|-----------|
| Atraso no desenvolvimento | Alto | Definir MVP e priorizar funcionalidades |
| Complexidade técnica | Médio | Utilizar tecnologias amplamente documentadas |
| Falta de tempo para o relatório | Alto | Produzir documentação ao longo do desenvolvimento |
| Alterações de requisitos | Médio | Manter backlog atualizado |

---

## 9. Development Methodology

O projeto será desenvolvido recorrendo à metodologia Agile Scrum, organizada em sprints de duas semanas.

No início de cada sprint serão definidas as funcionalidades a implementar e, no final, será realizada uma revisão dos resultados obtidos.

Esta abordagem permitirá uma evolução incremental da plataforma e uma adaptação contínua às necessidades identificadas durante o desenvolvimento.
